# Project Memory: 華山有田焼 Second Selection

This file is the project-specific memory entry for the Arita-yaki export platform. Prefer reading this first when working inside `/Users/salmonli/Downloads/有田烧出口平台`, then read `docs/PROJECT_BRIEF.md` only when deeper context is needed.

Li is building an overseas-facing Kazan / 華山 Arita-yaki Second Selection platform for overseas restaurant chefs, buyers, and high-end collectors. The business background is that Arita has many minor-flaw B-grade / second-selection pieces that still carry brand lineage, and the shop owner has authorized overseas sales under the Kazan brand.

The platform should not feel like a cheap outlet or generic ecommerce site. It should first build trust through culture, origin, brand history, vessel aesthetics, and transparent condition reporting, then guide inquiry or purchase.

Product direction:

- Default language is Japanese, with English and Chinese support.
- Phase 1 is a frontend demo, not backend implementation.
- Emphasize "authorized overseas sales partner", Arita/Kazan cultural communication, transparent Condition Report, and small-batch on-demand export.
- Long-term workflow: seller uploads phone photos -> AI generates multilingual descriptions and defect reports -> system estimates dimensions/shipping -> creates product cards -> buyer inquires or orders -> platform supports logistics/customs documents.

Design and behavior:

- Follow Arita Heritage / Museum Chronicle taste.
- Use Arita indigo, porcelain white, whitespace, and museum-like narrative.
- Avoid AI-template feeling, low-price marketplace feeling, and generic ecommerce patterns.
- Before major implementation decisions, confirm direction with the user.
- If there are risks such as shipping cost inversion, compliance problems, or brand dilution, state them directly and propose alternatives.
